# Activity And Runtime Evidence

Use this checklist to ensure tasks and reviews are evidence-driven.

## Activity Grounding

- Resolve non-trivial UiPath activities from official docs/MCP tools.
- Record package and activity names in the plan/task artifacts.

## Build Evidence

- Analyzer output JSON
- Test report(s)
- Package artifact path/version

## Runtime Evidence

- Runtime logs or execution output
- Resource verification outputs (queues/assets/bindings as applicable)
- Any required human approval evidence
